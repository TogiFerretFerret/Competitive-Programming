Let's first suppose that L=1L=1. As a first attempt, consider some sort of dynamic programming where for each index i≤Ni≤N, we count the number of ways to validly partition the first ii letters of the input string (i.e. the number of partitions so that each subsequence is a string MO⋯OMO⋯O with at most kk OO's). If we could compute this for all ii, then the value for i=Ni=N would be our answer. Unfortunately, it's not immediately obvious how to use the value at ii to help compute the value at i+1i+1, since if the letter at index i+1i+1 is an OO, then for any valid partition of the first ii letters, the number of ways to assign the new letter to an existing subsequence depends on how many of these subsequence have strictly less than kk OO's, which could vary based on the partition. We could try to augment the dynamic programming state with this information (e.g. for each mm, count the number of valid partitions where exactly mm subsequences have less than kk OO's), but computing these dynamic programming values would seem to require yet more information. We are looking for a roughly linear-time solution, so this approach does not seem promising.

Intuitively, the difficulty arises because for any particular prefix, each MM that we have seen so far can be in any of k+1k+1 different "states", depending on how many OO's have been assigned to it so far, and this affects the combinatorics of the unseen suffix, so the dynamic programming state in a forward-pass algorithm seemingly must include a histogram of how many MM's are in each state. This suggests reversing our perspective. In a back-to-front algorithm, where we count the number of valid partitions of each suffix, each OO can only be in one of two states (assigned to an MM, or not), so intuitively all that we need to update our counts is the number of unassigned OO's in the suffix. Moreover, this is always the total number of OO's in the suffix, minus kk times the total number of MM's in the suffix, irrespective of the partition!

More formally, suppose we have computed ViVi, the number of ways to partition the last ii letters so that all but one subsequence are strings MO⋯OMO⋯O with exactly kk OO's, and the remaining subsequence is the unused OO's. To compute Vi+1Vi+1 there are two cases. If the i+1i+1-st letter from the end is an OO, then Vi+1=ViVi+1=Vi since this letter cannot be used yet. Otherwise, Vi+1=(Oi−kMik)ViVi+1=(Oi−kMik)Vi, since for each valid partition of the last ii letters, there are (Oi−kMik)(Oi−kMik) ways to choose kk OO's from the unused OO's. Here, OiOi is the number of OO's in the last ii letters, and Mi=i−OiMi=i−Oi is the number of MM's in the last ii letters. This gives an O(N)O(N) time algorithm so long as we can efficiently compute the needed binomial coefficients. While this algorithm is most naturally expressed via a reverse pass through the input data, it can just as easily be implemented in a forward pass by maintaining the value Oi−kMiOi−kMi; see the code below.

There are several ways to compute the needed binomial coefficients efficiently. One way, which suffices for this problem, is to precompute all factorials up to N!N! (all modulo p=109+7p=109+7). We can then compute any given binomial coefficient (nk)=n!k!(n−k)!(nk)=n!k!(n−k)! efficiently by using Fermat's little theorem for the modular inverses: ap−2≡a−1modpap−2≡a−1modp for any 1≤a<p1≤a<p and prime pp. Since pp is large, we need to use fast exponentiation (i.e. repeated squaring) to compute the quantities ap−2modpap−2modp.

This is a complete solution for L=1L=1. To extend to general LL, it suffices to notice that no subsequence can cross between two different repeats of the input string: for example, if some subsequence starting in the second-to-last repeat used an OO in the last repeat, then the last repeat would not have enough OO's for all of its MM's; this argument generalizes. Thus, the answer for general LL is equal to the answer for L=1L=1, raised to the LL-th power. We can compute this quantity using fast exponentiation. 

#include <algorithm>
#include <iostream>
#include <string>
using namespace std;
#define MOD 1000000007
#define MAXN 1000001
 
int fact[MAXN];
 
int mul(int a, int b) { return (((long long)a) * b) % MOD; }
 
void computeFactorials() {
	fact[0] = 1;
	for (int i = 1; i <= MAXN; i++) fact[i] = mul(fact[i - 1], i);
}
 
int bexp(int a, long long e) {
	if (e == 0) return 1;
	int ans = bexp(a, e / 2);
	ans = mul(ans, ans);
	if (e % 2) ans = mul(ans, a);
	return ans;
}
 
int binv(int a) { return bexp(a, MOD - 2); }
 
int binom(int n, int k) {
	int ans = fact[n];
	ans = mul(ans, binv(fact[k]));
	ans = mul(ans, binv(fact[n - k]));
	return ans;
}
 
int main() {
	int N, K;
	long long L;
	cin >> K >> N >> L;
	computeFactorials();
	string s;
	cin >> s;
	int req = 0;
	int ans = 1;
	for (int i = 0; i < N; i++) {
		if (s[i] == 'M') {
			ans = mul(ans, binom(req + K, K));
			req += K;
		} else req--;
	}
	cout << bexp(ans, L) << '\n';
}
